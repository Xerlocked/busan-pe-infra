## 1. 프로젝트 개요

### 1.1 기술적 배경 및 당면 과제
- 현재 프론트엔드는 GitHub Pages(`xerlocked.github.io/pia/`) 기반의 프로토타입으로 구현되어 있으나, 채용 정보 업데이트 및 데이터 관리를 위한 백엔드/DB 레이어가 부재함.
- 공공기관 채용공고의 특성상 주로 **비정형 PDF 파일** 형태로 업로드되며, 내부의 표(Table)와 텍스트 구조가 기관마다 상이하여 100% 순수 코드로만 자동 파싱하기에는 예외 처리 공수가 극심함.
- 취업 준비와 병행하며 1년 동안 서비스를 안정적으로 유지해야 하므로, **운영 비용 최소화(무과금 지향)**와 **유지보수 효율성**을 동시에 만족하는 아키텍처가 필요함.

### 1.2 핵심 해결 방안
- **하이브리드 인프라:** 인프라 비용 부담이 없는 Cloudflare와 확장성이 뛰어난 AWS 서버리스(Serverless) 아키텍처를 결합.
- **반자동화 데이터 파이프라인:** PDF 텍스트 추출 라이브러리와 LLM(AI)을 결합하여, 채용공고 가공 업무를 자동화하되 최종 검수 단계를 두어 데이터 무결성을 확보함.

## 3. 핵심 기능 상세 정의

### 3.1 인프라 구성 및 웹서비스 호스팅 (AWS vs Cloudflare 조합)
- **Front-end Hosting (Cloudflare Pages):**
  - 빌드 및 배포가 간편하며, 대역폭(트래픽) 제한이 없어 대규모 유저 유입 시에도 추가 비용이 발생하지 않음.
  - 고유 브랜딩을 위한 커스텀 도메인 매핑 및 SSL 보안 인증서를 무료로 적용.
- **Back-end & DB (AWS Serverless):**
  - 24시간 구동되는 EC2 가상 서버 대신, 요청이 올 때만 실행되는 **AWS Lambda**를 채택하여 1년 운영 비용을 프리티어 범위 내(사실상 0원)로 방어.
  - 채용공고 스키마(항목) 변경에 유연하게 대처할 수 있도록 확장성이 뛰어난 NoSQL 데이터베이스인 **DynamoDB** 연동.

### 3.2 채용공고 파싱 및 데이터 반자동화 (Human-in-the-loop)
공공기관 채용공고 PDF를 시스템에 등록하고 정형 데이터로 변환하는 일련의 프로세스입니다.

{/* Reason: PDF 데이터 수집부터 DB 적재까지 단계별 유기적 연결 및 최종 검수 절차가 필수적이므로 시퀀스 컴포넌트 적용 */}
  <Step subtitle="관리자 액션" title="PDF 파일 수집 및 업로드">
    부산교통공사, 한국남부발전 등 타겟 기관에 인턴 및 신입 공고가 게시되면, 관리자가 해당 PDF 파일을 시스템(AWS S3 또는 백엔드 API)에 업로드합니다.
  </Step>
  <Step subtitle="AWS Lambda (Python)" title="1차 텍스트 및 레이아웃 추출">
    업로드 이벤트에 의해 Lambda 함수가 트리거되며, Python의 `pdfplumber` 라이브러리를 사용해 PDF 내부의 Raw 텍스트와 표(Table) 구조를 무손실 추출합니다.
  </Step>
  <Step subtitle="LLM API 연동" title="AI 기반 데이터 정형화 및 필터링">
    추출된 날것의 텍스트를 LLM 프롬프트에 입력하여 구조화된 JSON 데이터로 변환합니다.
    - **필터링 로직 내재화:** 프롬프트 조건 지정을 통해 해당 공고가 '체험형/채용형 인턴'인지 판별하는 플래그(`is_intern: true/false`)를 자동으로 생성하도록 유도합니다.
  </Step>
  <Step subtitle="웹 관리자 대시보드" title="최종 검수 및 DB 적재">
    AI가 파싱한 결과물([접수기간], [선발직렬], [자격요건], [인턴여부])을 관리자 화면에 UI 형태로 먼저 렌더링합니다. 기획자가 데이터 오매핑 여부를 최종 검수(10초 내외 스캔) 후 승인 버튼을 누르면 DynamoDB에 최종 반영됩니다.
  </Step>
</Sequence>
